# 调试可视化指南 (Debug Visualization Guide)

本指南旨在说明如何解读 PineappleHub 调试模式下显示的中间步骤图像。理解这些输出对于诊断光照、对焦或算法参数问题至关重要。

---

## 1. 平滑处理 (Smoothing)
**含义：** 应用高斯模糊 ($\sigma=1.0$) 后的原始图像。
*   **正常：** 略微模糊，但果眼的边缘仍然清晰可见。传感器噪点被抑制。
*   **调试：** 如果图像太模糊，可能是原始照片**失焦**。如果噪点太多，可能是相机的 ISO 设置过高。

## 2. 尺度校准 (Scale Calibration)
**含义：** 可视化被检测为“硬币”的物理参照物。
*   **正常：** 硬币被红框或高亮标出。
*   **调试：** 如果未检测到硬币（没有红色标记），后续所有的自适应参数都会失效。请检查硬币是否被遮挡或光照是否造成了强反光。

## 3. 纹理块/二值化 (Texture Patch / Binary)
**含义：** 应用自适应阈值后的二值图像。
*   **正常：** 呈现为一张噪点较多的“星图”。果眼中心显示为白色斑块，间隙显示为黑色。
*   **调试：**
    *   **全白**：对比度阈值 $C$ 过低（或为零）。
    *   **全黑**：对比度阈值 $C$ 过高。虽然 $C$ 现在是自动计算的，但这可能暗示光照过于平坦。

## 4. 二值融合 (Binary / Morphological Closing)
**含义：** 步骤 3 的“星图”经过闭运算处理。细小的噪点合并为实心的斑块。
*   **正常：** 明显的、大致呈圆形的斑块，代表单个果眼。
*   **调试：**
    *   **连成一片**：形态学半径 $R_{morph}$ 过大（硬币检测可能有误，导致比例尺偏大）。
    *   **过度破碎**：形态学半径 $R_{morph}$ 过小。

## 5. 形态学/ROI提取 (Morphology / ROI Extraction)
**含义：** 最终裁剪出的“果皮区域” (ROI)，仅包含：
1.  **Rotated ROI**: 经过旋转校正（扶正）后的原始 ROI。由于采用了**空域自适应滤波**，我们不再需要进行圆柱面展开。

*   **正常：**
    *   菠萝中轴线应垂直。
    *   图像应紧凑地包含大部分果眼区域。
*   **调试：**
    *   **未扶正**: 如果图像依然倾斜，说明 `MinAreaRect` 计算的角度有误。
    *   **空白**: 如果裁剪区域完全错误，说明纹理评分失败（可能视场内有其他高频纹理）。

## 6. 自适应响应图 (Adaptive Response Map)
**含义：** 经过 **空域自适应滤波 (LoG Kernel)** 处理后的增强图像。
*   **注意：** 这**不是**原始照片的裁剪图，而是一张**计算出的特征图 (Response Map)**。图中的亮度代表算法认为该处是果眼中心的置信度。
*   **视觉效果：** 背景（果肉）应被压暗为黑色。果眼中心（通常是深色的花腔）应显示为**高亮的白色光斑**。算法现在寻找的是“深色斑点”而非“亮斑”，以避免误检反光。
*   **解读：**
    *   **好**：清晰、分离的亮点，如同夜空中的星星，每个点对应一个独立的果眼。
    *   **坏（欠分割/粘连）**：亮斑呈现“虫状”、“条状”或像大脑回沟。这说明滤波器尺度 `Sigma` **过大**，导致相邻果眼融合。
    *   **坏（过分割/破碎）**：亮斑细碎、断裂，或者是仅仅是对反光点的响应。这说明滤波器尺度 `Sigma` **过小**，对噪声或局部反光过于敏感。
    *   **全黑**：光照校正（High-Pass）把信号减没了，或者 Sigma 估计过大。

## 7. 频域计数 (Frequency Analysis / Final Count)
**含义：** 最终计数结果。红叉标记检测到的果眼。
*   **调试：**
    *   **双重计数 (Double Counting)**：一个果眼上有两个叉。这通常被 V2.0 的 **物理NMS** (半径 12.5mm) 解决。如果出现，说明硬币尺寸估算严重偏小。
    *   **漏检 (Missed Counting)**：明显的果眼没有叉。可能是 **动态阈值** ($0.5 \times Max$) 太高了，或者响应图（步骤6）本身对比度不足。
